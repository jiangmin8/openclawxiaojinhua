# 巡检任务配置说明

## 任务概述
已创建两个定时巡检任务脚本，用于监控 PC 和路由器的系统状态。

## 创建的文件
1. `/home/lz/.openclaw/workspace/cron/pc_inspe` - PC巡检脚本
2. `/home/lz/.openclaw/workspace/cron/router_inspe` - 路由器巡检脚本
3. `/home/lz/.openclaw/workspace/cron/01-pc-in` - PC巡检 cron 配置
4. `/home/lz/.openclaw/workspace/cron/02-router-in` - 路由器巡检 cron 配置
5. `/home/lz/.openclaw/workspace/巡检任务说明.md` - 本说明文档

## 使用说明

### 1. 赋予执行权限
```bash
# WSL 终端执行
chmod +x /home/lz/.openclaw/workspace/cron/pc_inspe
chmod +x /home/lz/.openclaw/workspace/cron/router_inspe
```

### 2. 安装 cron 服务（如未安装）
```bash
sudo apt update
sudo apt install -y cron
sudo systemctl enable cron
sudo systemctl start cron
```

### 3. 安装并配置 ssh 密钥（如需要）
```bash
# 生成 SSH 密钥对（如果尚未生成）
ssh-keygen -t ed25519 -C "laptop@pc"

# 将公钥复制到路由器
ssh-copy-id root@192.168.8.1
```

### 4. 添加 cron 任务
```bash
# 将以下内容追加到 crontab
crontab -e
```

然后在 crontab 中添加：
```
# PC 巡检任务
/home/lz/.openclaw/workspace/cron/pc_inspe >> /var/log/cron/pc_inspe.log 2>&1

# 路由器巡检任务（需要先确保 SSH 免密登录已配置）
/home/lz/.openclaw/workspace/cron/router_inspe >> /var/log/cron/router_inspe.log 2>&1
```

### 5. 验证任务
```bash
# 立即执行一次测试
/home/lz/.openclaw/workspace/cron/pc_inspe

# 查看日志
tail -f /var/log/cron/pc_inspe.log
tail -f /var/log/cron/router_inspe.log
```

## 巡检内容

### PC巡检检查项
- **磁盘使用率**：使用率超过 80% 的分区
- **内存使用**：可用内存低于 4GB
- **Ollama 模型列表**：显示所有已安装模型
- **NVIDIA GPU 状态**：GPU 温度、显存使用情况
- **临时目录大文件**：/tmp 目录下最大的 5 个文件

### 路由器巡检检查项
- **运行时间**：系统 uptime
- **内存使用**：物理内存使用状况
- **SSD 挂载**：存储挂载状态和容量
- **AGH 状态**：音频处理硬件状态
- **Swap 使用**：交换分区使用情况
- **日志异常**：筛选 error/fail/warn 关键字

## 日志管理
- 所有日志保存在 `/tmp/` 目录下，7 天后自动清理
- PC 巡检日志示例：`/tmp/pc_inspe_20260508_102900.log`
- 路由器巡检日志示例：`/tmp/router_inspe_20260508_102900.log`
- 如不需要自动清理，可注释掉脚本中的清理命令

## ⚠️ 注意事项
1. 确保 SSH 免密登录已配置，否则路由器巡检会失败
2. 第一次执行前需要确保 cron 服务已运行
3. 检查 `/var/log/cron/` 目录的写入权限
4. 如日志文件过大，可定期手动清理 `/tmp/` 目录

## 常见问题排查

### 问题：crontab 报错 "No crontab for user"
**解决**：确保当前用户已登录过系统，执行 `crontab -l` 查看

### 问题：SSH 连接失败
**解决**：检查 SSH 服务是否开启，公钥是否正确配置
```bash
# 测试 SSH 连接
ssh root@192.168.8.1 uptime
```

### 问题：脚本权限不足
**解决**：执行 `chmod +x` 赋予执行权限
```bash
chmod +x /home/lz/.openclaw/workspace/cron/pc_inspe
chmod +x /home/lz/.openclaw/workspace/cron/router_inspe
```
