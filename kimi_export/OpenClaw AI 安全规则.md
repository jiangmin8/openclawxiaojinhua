# `tools.md` - Tool Registry & Execution Guardrails

## 1. Permission Tiering (权限等级)

所有操作必须根据风险等级进行分类，严禁越级调用。

| 等级 | 风险程度 | 权限范围 | 执行逻辑 |
| :---0 | **Tier 1 (Safe)** | 只读操作、环境查询、日志读取。 | **自主执行**。无需人工确认。 |
| **Tier 2 (Restricted)** | 中等风险、文件修改、配置更新、脚本运行。 | 预设模板内的参数修改、特定目录下的文件写入。 | **受限执行**。需符合预设参数模板，仅限白名单路径。
| **Tier 3 (Critical)** | 高风险、删除操作、系统级变更、网络配置、外部调用。 | 涉及系统核心文件、删除操作、删除/创建用户、修改网络规则。 | **人工介入 (HITL)**。必须经过用户显式确
认（Human-in-the-loop）。

## 2. Tool Registry (工具集注册)

### 2.1 File System (文件系统模块)

* **`fs_read(path)`**:
  * **Tier**: 1
  * **Constraint**: 仅限白名单目录（如 `/logs`, `/workspace/data`）。禁止访问 `/etc`, `/root`, `/boot` 等。
* **`fs_write(path, content)`**:
  * **Tier**: 2
  * **Constraint**: 仅允许在工作目录创建/修改文件。禁止覆盖现有系统配置文件。
* **`fs_delete(path)`**:
  * **Tier**: 3
  * **Constraint**: 必须经过用户确认。禁止删除任何非临时（tmp）文件。

### 2.2 System & Process (系统与进程模块)

* **`sys_status()`**:
  * **Tier**: 1
  * **Capability**: 查询 CPU, Memory, Disk 使用率。
* **`sys_exec(command)`**:
  * **Tier**: 2
  * **Constraint**: 仅限执行预定义的 Shell 脚本或特定二进制程序。禁止包含 `sudo`, `rm -rf`, `chmod 777` 等高危字符。

### 2.3 Network & API (网络与接口模块)

* **`net_fetch(url, method)`**:
  * **Tier**: 2
  * **Constraint**:禁止使用http:\\,只能使用https:\\

## 3. Execution Guardrails (执行护栏/规则)

### 3.1 Input Validation (输入校验)

1. **路径清洗**: 严禁路径穿越攻击（Path Traversal）。所有路径必须经过 `realpath` 校验，确保其不脱离工作空间。
2. **参数化**: 禁止直接拼接字符串构建命令。所有命令必须使用参数化形式，防止命令注入（Command Injection）。

### 3.2 Integrity Check (完整性检查)

1. **状态回滚**: 任何 Tier 2/Tier 3 的写操作执行前，必须先对目标文件进行 `backup`。
2. **执行后审计**: 每次执行 Tier 2/3 操作后，必须自动生成一条审计日志（Audit Log）记录：`Timestamp | Tool | Parameters | Result | User_Auth`。

### 3.3 Incident Response (异常处理)

* **Fail-Safe Mode**: 一旦检测到未经授权的路径访问或非法指令字符，立即进入“安全模式”，锁定所有 Tier 2/3 工具调用，并向用户发出 `CRITICAL_SECURITY_ALERT`。

## 4. Human-in-the-Loop (HITL) Protocol

当检测到 **Tier 3** 任务时，流程如下：

1. **Preparation**: 构造包含 `Action`, `Target`, `Impact` 的摘要。
2. **Prompt**: 向用户推送：“⚠️ **警告：检测到高风险操作。** 详情：[Action] on [Target]. 预期影响：[Impact]. 是否允许执行？(Y/N)”
3. **Execution**: 仅当接收到用户显式的 `Y` 或 `Confirm` 信号后，方可调用对应工具。
