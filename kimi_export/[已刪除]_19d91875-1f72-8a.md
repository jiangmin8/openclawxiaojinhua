## 一、OpenVPN 服务端部署（Ubuntu）

### 1. 安装 OpenVPN 和 Easy-RSA

```bash
sudo apt update
sudo apt install openvpn easy-rsa -y
```

[^65^][^67^]

### 2. 创建证书目录

```bash
make-cadir ~/openvpn-ca
cd ~/openvpn-ca
```

[^66^]

### 3. 初始化 PKI 并创建证书

```bash
# 初始化 PKI
./easyrsa init-pki

# 创建 CA 证书（无密码）
./easyrsa build-ca nopass

# 生成服务器证书
./easyrsa gen-req server nopass
./easyrsa sign server server

# 生成 Diffie-Hellman 参数
./easyrsa gen-dh

# 生成 TLS 认证密钥（可选，增强安全性）
openvpn --genkey secret ta.key
```

[^65^][^70^]

### 4. 复制证书到 OpenVPN 目录

```bash
sudo cp ~/openvpn-ca/pki/ca.crt /etc/openvpn/
sudo cp ~/openvpn-ca/pki/issued/server.crt /etc/openvpn/
sudo cp ~/openvpn-ca/pki/private/server.key /etc/openvpn/
sudo cp ~/openvpn-ca/pki/dh.pem /etc/openvpn/
sudo cp ~/openvpn-ca/ta.key /etc/openvpn/ 2>/dev/null || echo "ta.key 可选"
```

[^65^]

### 5. 创建服务端配置文件 `/etc/openvpn/server.conf`

```bash
sudo tee /etc/openvpn/server.conf << 'EOF'
port 1194
proto udp
dev tun

ca ca.crt
cert server.crt
key server.key
dh dh.pem

server 10.8.0.0 255.255.255.0
ifconfig-pool-persist ipp.txt

# 关键：推送默认网关，让所有流量走 VPN（透明代理）
push "redirect-gateway def1 bypass-dhcp"
push "dhcp-option DNS 8.8.8.8"
push "dhcp-option DNS 8.8.4.4"

keepalive 10 120
tls-auth ta.key 0  # 如果生成了 ta.key，否则注释掉
cipher AES-256-GCM
persist-key
persist-tun

status openvpn-status.log
verb 3
EOF
```

[^65^][^67^][^72^]

### 6. 启用 IP 转发和 NAT（关键步骤）

```bash
# 启用 IP 转发
echo "net.ipv4.ip_forward = 1" | sudo tee /etc/sysctl.conf
sudo sysctl -p

# 配置 NAT（将 VPN 客户端流量转发到互联网）
# 假设你的公网接口是 eth0（用 ip addr 查看实际接口名）
sudo iptables -t nat -A POSTROUTING -s 10.8.0.0/24 -o eth0 -j MASQUERADE

# 保存 iptables 规则（Ubuntu 22.04+）
sudo apt install iptables-persistent -y
sudo netfilter-persistent save
```

[^65^][^68^]

### 7. 启动 OpenVPN 服务

```bash
sudo systemctl start openvpn@server
sudo systemctl enable openvpn@server

# 查看状态
sudo systemctl status openvpn@server
sudo tail -f /var/log/syslog | grep openvpn
```

## 二、OpenVPN 客户端配置（Ubuntu）

### 1. 安装客户端

```bash
sudo apt update
sudo apt install openvpn -y
```

[^71^][^76^]

### 2. 获取客户端证书

在服务端生成客户端证书：

```bash
cd ~/openvpn-ca
./easyrsa gen-req client1 nopass
./easyrsa sign client client1
```

[^65^][^70^]

### 3. 创建客户端配置文件 `client1.ovpn`

```bash
client
dev tun
proto udp
remote 你的服务器IP 1194
resolv-retry infinite
nobind
persist-key
persist-tun

# 关键：让所有流量通过 VPN（透明代理）
redirect-gateway def1

remote-cert-tls server
cipher AES-256-GCM
verb 3

# 证书内嵌方式（推荐）
<ca>
[粘贴 ca.crt 内容]
</ca>

<cert>
[粘贴 client1.crt 内容]
</ca>

<key>
[粘贴 client1.key 内容]
</key>

<tls-auth>
[粘贴 ta.key 内容]
</tls-auth>
key-direction 1
```

[^65^][^74^][^75^]

### 4. 免密码登录（可选）

如果服务端需要用户名密码认证：

```bash
# 创建密码文件
echo -e "用户名\n密码" | sudo tee /etc/openvpn/pass.txt
sudo chmod 600 /etc/openvpn/pass.txt

# 在 client.ovpn 中添加
auth-user-pass /etc/openvpn/pass.txt
```

[^74^][^76^]

### 5. 启动客户端连接

```bash
# 临时测试连接
sudo openvpn --config /path/to/client1.ovpn

# 或作为系统服务运行
sudo cp client1.ovpn /etc/openvpn/client.conf
sudo systemctl start openvpn@client
sudo systemctl enable openvpn@client
```
