## 拓扑位置决定角色

```
【软路由主路由】
光猫 → [软路由拨号] → 硬路由(AP模式) → 设备
        ↑
    网络核心，所有流量必经

【旁路由】
光猫 → [硬路由拨号] → 设备
              ↘
              [软路由旁挂]
                ↑
          只有指定设备走它
```

---

## 关键区别

| | 软路由主路由 | 旁路由 |
|--|------------|--------|
| **位置** | 光猫和硬路由之间 | 硬路由后面（旁挂） |
| **是否必须** | 没它就没网 | 没它也能上网 |
| **谁拨号** | 软路由 | 硬路由 |
| **控制力度** | 全局强制 | 按需选择 |

---

## 旁路由的典型接法

```
硬路由 LAN口 ──┬── 普通设备（网关指向硬路由，直连）
              │
              └── 软路由 WAN口
                      ↓
                   软路由 LAN口 ── 特殊设备（网关指向软路由）
```

或者更常见的**单线复用**：
- 软路由只接一个LAN口，硬路由开DHCP但把网关改成软路由IP

---

## 为什么选旁路由？

| 场景 | 原因 |
|-----|------|
| 硬路由WiFi很好，不想换 | 保留原厂固件和WiFi性能 |
| 家人要稳定，自己要折腾 | 不影响他人，自己设备改网关就行 |
| 软路由性能弱/不稳定 | 硬路由保底，软路由崩了也能上网 |

你的网络环境是啥？硬路由型号说说，帮你看看哪种接法最省事。

## 旁路由的真实用途

你说"防御"其实只占了很小一部分，更多人用旁路由是为了：

- **去广告**（AdGuard Home）
- **科学上网**（Clash/PassWall）
- **内网穿透**（frp/nps）
- **Docker跑服务**（Home Assistant等）

**防火墙/防御**反而是旁路由相对弱的一环，因为：
- 它只处理"愿意走它"的流量
- 设备改回自动获取网关就能绕开它
- 真正的安全防御需要主路由或独立防火墙

## 软路由（Software Router）是什么？

软路由是**用通用计算设备（PC/工控机/树莓派等）配合软件系统来实现路由功能**，它**取代或升级了传统硬件路由器的核心功能**，而不是在旁边"辅助"。

### 核心区别

| 传统硬路由 | 软路由 |
|-----------|--------|
| 专用芯片处理路由功能 | CPU/软件处理路由功能 |
| 功能固定，扩展性差 | 功能灵活，可安装插件 |
| 性能受限于硬件规格 | 性能取决于CPU/内存配置 |

### 常见使用方式

1. **主路由模式**：软路由直接拨号上网，硬路由当AP用
2. **旁路由模式**：软路由作为网关，硬路由负责WiFi，流量可选择性经过软路由处理

### 你说的"防火墙处理异常"更接近**旁路由（旁路网关）**的用法：

```
[光猫] → [硬路由拨号] → [软路由作为旁路网关] → 设备
                           ↓
                    广告过滤/科学上网/防火墙规则
```

设备手动设置网关指向软路由，特定流量（如国外网站）走软路由处理，其他直连。

### 软路由的优势

- **OpenWrt/iStoreOS** 等系统可装各种插件
- 去广告、流量控制、多拨、Docker、科学上网等
- 性能更强，适合千兆以上宽带

你具体是想了解旁路由的配置，还是在考虑入手软路由？
