## 具体配置步骤（对接你现有的 Ollama）

### 1. 确认 Ollama 监听地址（WSL2 里执行）

```bash
# 查看 Ollama 监听状态
ss -tlnp | grep 11434
# 或
curl http://localhost:11434/api/tags
```

如果只在 `127.0.0.1:11434`，需要改成监听所有接口，否则 Docker 容器访问不到：

```bash
# 临时生效
export OLLAMA_HOST=0.0.0.0:11434
ollama serve &

# 或永久修改（加到 ~/.bashrc）
echo 'export OLLAMA_HOST=0.0.0.0:11434' >> ~/.bashrc
```

### 2. 获取 WSL2 的 IP（给 Docker 容器用）

```bash
ip addr show eth0 | grep 'inet ' | awk '{print $2}' | cut -d/ -f1
# 输出类似：172.19.32.1（这个 IP 每次 WSL2 重启会变，后面会说固定方法）
```

### 3. 修改 Bytebot 的 LiteLLM 配置

找到你下载的 Bytebot 目录里的这个文件：

```
packages/bytebot-llm-proxy/litellm-config.yaml
```

改成：

```yaml
model_list:
  - model_name: qwen-local          # Bytebot 里用的名字
    litellm_params:
      model: ollama/qwen2.5:14b     # 你 WSL2 里已有的模型
      api_base: http://172.19.32.1:11434  # WSL2 IP，按你实际改的填
      timeout: 120                   # 本地模型慢，给多点时间

  - model_name: gemma-local
    litellm_params:
      model: ollama/gemma4:e4b
      api_base: http://172.19.32.1:11434
```

### 4. 启动 Bytebot（带 LLM 代理）

```bash
cd bytebot-main
docker compose up bytebot-llm-proxy bytebot -d
```

### 5. 验证连通性

```bash
# 进 Bytebot 容器测试
docker exec -it bytebot-llm-proxy curl http://172.19.32.1:11434/api/tags
```

能返回模型列表就说明通了。

## 一个坑：WSL2 IP 会变

每次重启 WSL2，IP 可能变。两个解决办法：

**办法 A：固定 IP（推荐）**

在 Windows PowerShell（管理员）执行：

```powershell
# 查看 WSL 虚拟网卡
wsl hostname -I
# 然后在 .wslconfig 里固定（C:\Users\lz\.wslconfig）
```

```ini
# C:\Users\lz\.wslconfig
[wsl2]
networkingMode=mirrored  # Win11 支持，让 WSL2 和 Win 共享 localhost
dnsTunneling=true
firewall=true
autoProxy=true
```

开了 `mirrored` 模式后，Bytebot 容器里可以直接用 `host.docker.internal:11434` 访问 WSL2 的 Ollama，不用记 IP。

**办法 B：用 host 网络模式（简单但粗暴）**

启动 Bytebot 时加 `--network host`，让容器和宿主机共享网络栈，直接 `localhost:11434` 就能通。但 Docker Desktop Win 的 host 模式支持有限。

## Bytebot 连接本地模型的两种方式

### 方式一：直接配置 Ollama（最简单）

在 `docker/.env` 或 LiteLLM 配置文件中添加：

```yaml
# packages/bytebot-llm-proxy/litellm-config.yaml
model_list:
  - model_name: local-qwen
    litellm_params:
      model: ollama/qwen2.5:14b
      api_base: http://host.docker.internal:11434  # Windows/Mac
      # 或 http://你的WSL2_IP:11434  # 如果你Ollama在WSL2
```

然后设置环境变量让 Bytebot 走代理：
```bash
BYTEBOT_LLM_PROXY_URL=http://bytebot-llm-proxy:4000
```

### 方式二：Docker Compose 内嵌 Ollama

如果你希望 Ollama 也容器化，可以在 `docker-compose.yml` 里加一个 `ollama` 服务，让 Bytebot 容器网络直接访问 `http://ollama:11434`。
