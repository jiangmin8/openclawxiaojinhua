我来帮你整理 Cloudflare 状态 API 的文档。这是一个结构化的 REST API，用于程序化访问 Cloudflare 的服务状态页面。

## Cloudflare 状态 API 文档

### 基础信息
- **文档用途**：允许用户通过程序访问 Cloudflare 状态页面的各个元素
- **基础域名**：`https://www.cloudflarestatus.com/api/v2/`

---

### 数据端点

#### 1. 摘要 (Summary)
获取状态页面完整摘要，包括状态指示器、组件状态、未解决事件以及即将进行或正在进行的计划维护。

```
GET https://www.cloudflarestatus.com/api/v2/summary.json
```

#### 2. 现状 (Status)
获取整页的状态汇总，包含状态指标和人工描述。

**状态指标**：无、次要、主要、关键  
**描述示例**："所有系统正常运行"、"部分系统故障"、"重大服务中断"

```
GET https://www.cloudflarestatus.com/api/v2/status.json
```

#### 3. 组成部分 (Components)
获取页面的所有组件及其状态。

**组件状态**：
- `operational` - 运行中
- `degraded_performance` - 性能下降
- `partial_outage` - 部分中断
- `major_outage` - 重大中断

```
GET https://www.cloudflarestatus.com/api/v2/components.json
```

---

### 事故 (Incidents)

事件是状态页面的核心，由多个事件更新组成。

**事件状态**：调查中 → 已确认 → 监控中 → 解决 → 尸检  
**影响级别**：无（黑色）、轻微（黄色）、重大（橙色）、关键（红色）

| 端点 | 说明 | URL |
|------|------|-----|
| 未解决事件 | 只返回调查中、识别或监控状态的事件 | `/incidents/unresolved.json` |
| 所有事件 | 最近50起事件（包括已解决和尸检状态） | `/incidents.json` |

```
GET https://www.cloudflarestatus.com/api/v2/incidents/unresolved.json
GET https://www.cloudflarestatus.com/api/v2/incidents.json
```

---

### 定期维护 (Scheduled Maintenances)

计划中的停机、升级或基础设施施工通知。

**维护状态**：预约 → 进行中 → 验证 → 完成  
**影响级别**：同事故（无、轻微、重大、关键）

| 端点 | 说明 | URL |
|------|------|-----|
| 即将进行的维护 | 只返回计划状态的维护 | `/scheduled-maintenances/upcoming.json` |
| 活跃的维护 | 进行中或验证状态的维护 | `/scheduled-maintenances/active.json` |
| 所有维护 | 最近50个维护（包括已完成） | `/scheduled-maintenances.json` |

```
GET https://www.cloudflarestatus.com/api/v2/scheduled-maintenances/upcoming.json
GET https://www.cloudflarestatus.com/api/v2/scheduled-maintenances/active.json
GET https://www.cloudflarestatus.com/api/v2/scheduled-maintenances.json
```

---

```html
<script src="https://cdn.statuspage.io/se-v2.js"></script>
```
