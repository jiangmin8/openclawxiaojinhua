## WARP 的 DNS 工作方式

| 场景 | DNS 行为 |
|------|---------|
| **WARP 开启（正常模式）** | WARP 劫持所有 DNS → 通过加密隧道（DoH/DoT）发送到 Cloudflare |

| **WARP 关闭** | 使用系统设置的 DNS（你配的 `2606:4700:4700::1111`） |

---

## 关键区别

| 配置 | 作用范围 |
|------|---------|
| **Windows 系统 DNS** | WARP 关闭时生效 |
| **WARP 内置 DNS** | WARP 开启时覆盖系统设置 |

- **WARP 开启时**：系统 DNS 配置被忽略，WARP 强制走 `1.1.1.1` / `2606:4700:4700::1111`
- **WARP 关闭时**：系统 DNS 生效，走你配置的 Cloudflare IPv6 DNS

我看到问题了！**WARP 显示 "DNS 查找失败"**，这是 WARP 本身的错误，不是你的系统 DNS 配置问题。

## 问题根源

| 现象 | 说明 |
|------|------|
| `nslookup` 指向 `2606:4700:4700::1111` | 系统 DNS 配置正确 ✅ |
| WARP 报错 "DNS 查找失败" | **WARP 无法连接到 Cloudflare 的 DoH/DoT** |
| 请求超时 | WARP 隧道未建立，但 DNS 已被劫持到 WARP |

## 常见原因

1. **WARP 被墙或干扰** — 中国大陆常见，CF_DNS_LOOKUP_FAILURE 是典型症状
2. **本地 53 端口被占用** — 其他 DNS 软件冲突
3. **WARP 版本问题** — 需要更新或换版本

---

## 解决方案

### 方案 1：切换 WARP 模式（最快）

在 WARP 客户端里：
- **关闭 WARP**（只用 1.1.1.1 DNS，不代理流量）
- 或切换为 **"仅 DNS" 模式**

### 方案 2：检查端口占用

```powershell
# 查看 53 端口是否被占用
netstat -ano | findstr ":53"
```

### 方案 3：换用 WARP Zero Trust 或手动 DoH

如果 WARP 被干扰，可以：
- 用 **Cloudflare Zero Trust** 替代
- 或系统直接设置 DoH（不用 WARP 客户端）

---

## 你的选择

| 需求 | 建议 |
|------|------|
| 只想用 Cloudflare DNS | **关闭 WARP**，系统 DNS 已设置好 `2606:4700:4700::1111` |
| 需要 WARP 代理功能 | 尝试 **WARP Zero Trust** 或 **WireGuard + Cloudflare** |
| 两者都要 | 换网络环境（如手机热点）测试 WARP 是否正常 |

---

**你现在需要 WARP 的代理功能，还是只要 DNS 就行？**  
如果只要 DNS，**直接退出 WARP**，系统已经配置好了。
