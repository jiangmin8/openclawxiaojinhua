**W11 版就行**，不用折腾 Linux 版。

---

## 理由

| 对比 | W11 版 | Linux 版 |
|---|---|---|
| 你的使用场景 | 人在 Windows，OpenClaw 在 WSL | 纯 Linux 服务器 |
| 安装 | 双击 exe，完事 | 还要在 WSL 里跑脚本、配服务 |
| 日常使用 | 直接点图标启动 | 得开 WSL 终端、起服务 |
| 网络连接 | WSL IP 或 localhost 转发（上面已给方案） | 同机，无网络问题 |

你的 OpenClaw 在 WSL 里只是**提供 API 服务**，ClawPanel 作为**客户端**连过去就行。W11 桌面版体验更好，没必要把面板也塞进 WSL。

---

## 唯一选 Linux 版的情况

- 你打算 **24h 开机、远程访问**（比如公司电脑当服务器，家里连）
- 或者你 **根本不用 Windows 桌面**，纯命令行操作

否则 W11 版 + WSL OpenClaw 是最顺的。

## 步骤

### 1. WSL 侧：让 OpenClaw 监听所有接口

编辑你的 OpenClaw 配置文件（通常是 `~/.openclaw/config.yaml` 或 `~/.openclaw/config.json`）：

```yaml
server:
  host: "0.0.0.0"   # 原来是 127.0.0.1，改成这个
  port: 8080
```

然后重启 OpenClaw。

> 如果你不确定配置文件位置，在 WSL 里跑：
> ```bash
> find ~ -name "config.yaml" -path "*openclaw*" 2>/dev/null
> ```

### 2. 获取 WSL 的 IP 地址

在 WSL 终端里：

```bash
ip addr show eth0 | grep 'inet ' | awk '{print $2}' | cut -d/ -f1
```

假设输出是 `172.24.128.2`，那么 OpenClaw 的地址就是 `http://172.24.128.2:8080`

### 3. Windows 侧 ClawPanel 里配置

打开 ClawPanel → **模型配置** → 添加自定义服务商：

- **Base URL**: `http://<WSL_IP>:8080` （比如 `http://172.24.128.2:8080`）
- **API Key**: 填你 OpenClaw 的 key（如果有的话）

测试连通，保存即可。

---

## 一个坑：WSL IP 会变

每次 WSL 重启后 IP 可能变化。两个解决办法：

**办法 A（推荐）：用 localhost 转发**

在 Windows PowerShell（管理员）里执行一次：

```powershell
netsh interface portproxy add v4tov4 listenport=8080 listenaddress=127.0.0.1 connectport=8080 connectaddress=<WSL_IP>
```

然后 ClawPanel 里直接填 `http://127.0.0.1:8080`，永远不变。

**办法 B：固定 WSL IP**

在 `C:\Users\lz\.wslconfig` 里写：

```ini
[wsl2]
networkingMode=mirrored
```

这样 WSL 和 Windows 共享网络栈，WSL 服务直接通过 `localhost` 访问，不需要记 IP。需要 WSL 版本 2.0.0+。

## 方案一：ClawPanel Pro（已有 OpenClaw 环境）

这是官方主推路线，ClawPanel 作为纯面板接管你已有的 OpenClaw。

**步骤：**
1. 安装 ClawPanel Pro（桌面版或 Web 版均可）
2. 首次打开时，ClawPanel 会自动检测系统中的 OpenClaw 配置
3. 进入 **模型配置** 页面，添加你的 AI 服务商（DeepSeek / OpenAI / 阿里通义等），填好 Base URL 和 API Key，测试连通后保存 [^2^]
4. 进入 **服务管理** 页面，点击「启动」Gateway
5. 去 **实时聊天** 页面发一句测试，确认链路打通

如果是 Linux 服务器无桌面环境，用一键脚本部署 Web 版：
```bash
curl -fsSL https://raw.githubusercontent.com/qingchencloud/clawpanel/main/scripts/linux-deploy.sh | bash
```
部署完访问 `http://服务器IP:1420` [^2^]

## 方案二：ClawPanel Lite（内置 OpenClaw，开箱即用）

适合没有现成 OpenClaw 环境、想一步到位的新手。

Lite 内置了 OpenClaw 运行时，安装完直接可用，不需要额外链接步骤 [^3^]。

Linux 一键安装：
```bash

```
