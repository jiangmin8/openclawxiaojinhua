## 错误信息解读

```
LLM request failed: network connection error
```

**问题**：OpenClaw 调用 Ollama 时网络连接失败

**可能原因**：
1. Ollama 没启动
2. Ollama 端口被占用或配置错误
3. OpenClaw 配置的 Ollama 地址不对

---

## 快速排查

### 1. 检查 Ollama 是否运行
```bash
# 在命令行输入
ollama list
```
- 如果显示模型列表 → Ollama 正常
- 如果报错 → Ollama 没启动，输入 `ollama serve` 启动

### 2. 检查 OpenClaw 的 Ollama 配置
```bash
# 查看当前配置的模型地址
openclaw config get llm.ollama.url
```
- 应该是 `http://localhost:11434` 或 `http://127.0.0.1:11434`

### 3. 测试 Ollama 端口
```bash
curl http://localhost:11434/api/tags
```
- 有返回 JSON → 正常
- 没返回 → 防火墙或端口问题

---

## 最可能的解决方案

**Ollama 没开机自启**，手动启动：
```bash
ollama serve
```

或者在后台运行：
```bash
ollama serve &
```

---

## 设置开机自启（一劳永逸）

**Windows**：
```powershell
# 按 Win+R，输入
shell:startup
# 把 Ollama 快捷方式放进去
```

**Linux**：
```bash
sudo systemctl enable ollama
```
