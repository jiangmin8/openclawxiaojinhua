`qwen2.5-coder:7b` 是写代码的，对话能力很弱。你需要换 `qwen2.5:7b`（普通聊天版）。

**请依次执行：**

1. **在 WSL 里下载聊天版模型**（如果还没下）：
   ```bash
   ollama pull qwen2.5:7b
   ```

2. **在 UI 的 Provider 设置里添加新模型**（跟之前加 `qwen2.5-coder:7b` 一样的方式）：
   - 模型 ID 填：`qwen2.5:7b`
   - supportsTools 打勾
   - supportsUsageInStreaming 打勾

3. **刷新页面（Ctrl+F5）**，然后在顶部模型下拉菜单选 `qwen2.5:7b`

4. **测试对话**——这个模型应该能正常聊天了，不会像 coder 版那样"自闭"。

如果你已经执行了 `ollama pull qwen2.5:7b`，直接做第 2 步就行。需要我帮你确认当前 Ollama 里有哪些模型可用吗？
