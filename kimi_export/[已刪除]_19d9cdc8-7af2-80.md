这种情况通常是因为小米路由器的管理IP和上级路由冲突，或者它获取的是内网IP而非公网IP。让我帮你排查一下：

## 快速诊断步骤

### 1. 确认小米路由器的实际管理IP

在连接了小米路由器WiFi的设备上执行：

```bash
# 查看默认网关（即小米路由器的LAN IP）
ip route | grep default
# 或
ipconfig | findstr "Gateway"
```

### 2. 检查IP冲突

你的网络层级多，容易IP段冲突。常见情况：
- GL.iNet 默认可能是 `192.168.8.1`
- AWT路由器 默认可能是 `192.168.1.1` 或 `192.168.0.1`
- 小米AX3000 默认是 `192.168.31.1`

**如果上级路由也是 `192.168.31.x` 网段，就会冲突导致无法访问。**

### 3. 解决方案

**方案A：修改小米路由器IP段（推荐）**

1. 先断开小米路由器与AWT的连接（只保留小米的WiFi）
2. 电脑连接小米WiFi，访问 `192.168.31.1` 或 `miwifi.com`
3. 登录后进入 **常用设置 → 局域网设置(LAN)**
4. 将IP地址改为 `192.168.50.1`（或其他不冲突的网段）
5. 保存重启，再接回上级网络

**方案B：通过GL.iNet查看小米获取的IP**

登录GL.iNet管理界面，查看DHCP客户端列表，找到小米路由器获取的WAN IP，尝试用那个IP+小米端口访问（部分小米路由器支持 `http://<IP>:8080`）。

### 4. 如果以上都不行

尝试小米路由器的**恢复模式**：
- 断电 → 按住Reset键 → 通电保持10秒 → 指示灯变蓝/黄
- 电脑有线连接小米LAN口，访问 `192.168.31.1`
