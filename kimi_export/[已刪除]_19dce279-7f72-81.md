`curl` 通过管道 `| sh` 的方式执行时，**不支持断点续传**。如果下载过程中断，整个安装脚本会从头开始重新下载和执行。

---

### 解决方案

**方案一：先下载脚本到本地，再执行（推荐）**

```bash
# 1. 下载安装脚本到本地（支持断点续传 -C -）
curl -fsSL -C - -o install.sh https://ollama.com/install.sh

# 2. 给执行权限
chmod +x install.sh

# 3. 执行安装
sh install.sh
```

如果下载中断，重新执行第一步的 `curl -C -` 命令即可从断点继续下载脚本文件。

---

**方案二：先解决依赖，再重新执行原命令**

既然你已经安装了 `zstd`，网络稳定的情况下直接重新运行原命令即可，脚本本身不大，重新下载很快：

```bash
curl -fsSL https://ollama.com/install.sh | sh
```

---

**关键区别**：

| 方式 | 断点续传 | 适用场景 |
|------|---------|---------|
| `curl \| sh` | ❌ 不支持 | 网络稳定时快速安装 |
| `curl -o file && sh file` | ✅ 支持 | 网络不稳定或需要反复调试 |

建议用 **方案一**，更稳妥。

更新 WSL 及其中的 Ubuntu 系统，需要分别在 **Windows 侧** 和 **WSL 内部** 执行命令。以下是完整流程：

---

### 一、更新 WSL 本身（Windows 侧）

以管理员身份打开 **PowerShell** 或 **Windows Terminal**，执行：

```powershell
# 更新 WSL 内核到最新版本
wsl --update

# 更新完成后重启 WSL 以生效
wsl --shutdown
```

根据 Microsoft 官方文档，新安装的发行版默认使用 WSL 2，你也可以通过 `wsl --set-version <发行版名称> 2` 来切换版本 [^8^]。

---

### 二、更新 Ubuntu 系统包（WSL 内部）

进入 Ubuntu 终端后，依次执行：

```bash
# 1. 刷新软件源列表
sudo apt update

# 2. 升级已安装的软件包
sudo apt upgrade -y

# 3. 完整升级（处理依赖关系变更）
sudo apt full-upgrade -y

# 4. 清理不再需要的旧包，释放空间
sudo apt autoremove -y
sudo apt clean
```

这是 Ubuntu/Debian 系发行版的标准维护流程 [^7^][^1^]。

---

### 三、（可选）升级 Ubuntu 到下一个大版本

如果你想把 Ubuntu 从 22.04 升级到 24.04，或从 24.04 升级到 26.04，先确保当前系统已完全更新，然后：

```bash
# 安装升级工具
sudo apt install update-manager-core

# 编辑配置文件，确认升级策略（通常保持默认即可）
sudo nano /etc/update-manager/release-upgrades
# 确保包含：Prompt=lts

# 执行版本升级
sudo do-release-upgrade
```

升级过程需要交互确认，耗时较长，建议提前备份重要数据 [^13^][^16^]。

---

### 快速一键命令

如果你只想快速完成日常更新，可以直接在 Ubuntu 内运行：

```bash
sudo apt update && sudo apt upgrade -y && sudo apt autoremove -y
```

这样即可完成软件包的常规维护。
