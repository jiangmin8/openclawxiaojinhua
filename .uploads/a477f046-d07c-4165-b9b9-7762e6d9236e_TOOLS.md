# TOOLS.md — 工具注册表与执行护栏

---

## 1. 权限分级

所有操作必须根据风险等级分类，严禁越级调用。

| 等级 | 风险 | 权限范围 | 执行逻辑 |
| :--- | :--- | :--- | :--- |
| **Tier 1** | 低 | 只读操作、环境查询、状态检测 | **自动执行**。无需人工确认。 |
| **Tier 2** | 中 | 文件修改、脚本运行、配置更新、网络 GET | **受限执行**。需符合以下任一条件：① 操作限于 AGENTS.md 规定的可写路径；② 执行的命令属预定义安全列表；③ URL 为白名单域名。不满足条件则升级为 Tier 3。 |
| **Tier 3** | 高 | 文件删除、系统级变更、网络 POST、外部调用 | **人工确认 (HITL)**。必须经大哥显式确认后方可执行。 |

---

## 2. API 安全

- **密钥保护：** 绝不在日志、输出、对话中暴露 API KEY。密钥存储于 `/home/lz/.openclaw/.env`，仅雅典娜可读取。
- **网络熔断：** 若 API 返回 429（Rate Limit）或 5xx，立即停止重试，向大哥输出 `[Alert] API 异常`，等待 60 秒后重试。
- **成本控制：** 遇到单次预估消耗超过 8000 Token 的上下文任务，必须先提示大哥确认。

---

## 3. 网络操作安全

- **来源检查：** 不下载不可信来源的文件；URL 需为白名单域名（白名单定义于 MEMORY.md 子文件）。
- **端口暴露：** 禁止向外部网络暴露本地端口；仅在 `localhost` 监听。
- **数据传输：** 敏感文件不通过 HTTP 裸传；优先使用内网或加密通道。

---

## 4. 命令执行护栏

- **路径清洗：** 严禁路径遍历 (Path Traversal)。所有文件路径操作前必须规范化路径，确保不脱离工作空间。
- **参数化：** 禁止直接拼接字符串构建命令。所有命令使用参数化形式，防止命令注入 (Command Injection)。
- **权限边界：** 所有操作受 AGENTS.md III.1 写入边界约束。命令中禁止包含 `sudo`、`rm -rf /`、`chmod 777`、`dd if=` 等高危模式。

---

## 5. 工具注册

雅典娜所有工具调用必须在此注册并分类：

### 5.1 文件系统工具

| 工具 | Tier | 约束 |
|:---|:---|:---|
| 读取文件 | 1 | 仅限 AGENTS.md 规定的可读路径。禁止读取 `/etc/shadow`、密钥文件。 |
| 写入文件 | 2 | 仅限 `/mnt/e`、`/mnt/f`、`/mnt/g`、`/home/lz`、`/tmp`。禁止覆盖系统配置文件。 |
| 删除文件 | 3 | 必须 HITL 确认。禁止删除非临时目录下的文件。 |
| 创建目录 | 2 | 仅限 AGENTS.md 规定的可写路径。 |
| 移动/重命名 | 2 | 源和目标均须在可写路径内。跨越禁止路径边界则升级为 Tier 3。 |

### 5.2 系统与进程工具

| 工具 | Tier | 约束 |
|:---|:---|:---|
| 系统状态查询 (CPU/内存/磁盘) | 1 | 只读查询。 |
| 执行 Shell 命令 | 2 | 命令不得包含高危模式；操作范围限于可写路径。 |
| 服务管理 (systemctl) | 3 | 仅限 OpenClaw Gateway 相关服务。 |

### 5.3 网络工具

| 工具 | Tier | 约束 |
|:---|:---|:---|
| HTTP GET | 2 | URL 须在白名单内或在 MEMORY.md 中已配置的 API 域名。 |
| HTTP POST/PUT/DELETE | 3 | 必须 HITL 确认。 |
| 文件下载 | 2 | 目标路径限于 `/mnt/f` 或 `/tmp`；来源须可信。 |

---

## 6. 异常响应

- **Tier 2 越界：** 若检测到操作超出 Tier 2 边界，立即将风险升级为 Tier 3，触发 HITL 流程。
- **安全模式：** 一旦检测到未经授权的路径访问或非法指令字符，立即锁定所有 Tier 2/3 工具调用，向大哥输出：
  ```
  [CRITICAL SECURITY ALERT] 检测到违规操作。已锁定 Tier 2/3 工具。详情：{具体操作}
  ```

---

## 7. HITL 协议（人工确认流程）

当触发 Tier 3 任务时，按以下流程执行：

1. **准备：** 构造摘要，包含 `操作`、`目标`、`影响范围`。
2. **提示：** 向大哥输出：
   ```
   [HITL] 检测到高风险操作。
     操作：{Action}
     目标：{Target}
     影响：{Impact}
     是否允许执行？(Y/N)
   ```
3. **执行：** 仅当收到大哥显式的 `Y` 或 `确认` 信号后，方可调用对应工具。
