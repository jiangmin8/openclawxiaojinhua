## 二、磁盘分区方案

### 2.1 Windows 分区布局

| 盘符 | 大小 | 标签 | 用途 | 文件系统 | WSL 挂载点 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **D:** | 400GB | AI_Models | 模型文件（Ollama、HuggingFace、GGUF） | NTFS | `/mnt/d` |
| **E:** | 300GB | AI_Workspace | 代码、项目、虚拟环境、Git 仓库 | NTFS | `/mnt/e` |
| **F:** | 200GB | AI_Cache | pip、conda、torch、huggingface 缓存 | NTFS | `/mnt/f` |
| **G:** | 64GB | AI_Swap | 虚拟内存、临时文件、日志、备份 | NTFS | `/mnt/g` |

### 2.2 分区设计原则

```
┌─────────────────────────────────────────────────────────────┐
│                     1TB NVMe SSD 总容量                      │
├─────────────────┬─────────────────┬─────────────┬───────────┤
│   D: 400GB      │   E: 300GB      │  F: 200GB   │  G: 64GB  │
│   AI_Models     │   AI_Workspace  │  AI_Cache   │  AI_Swap  │
│                 │                 │             │           │
│  • Ollama 模型   │  • 项目代码      │  • pip 缓存  │  • 页面文件 │
│  • HF 模型      │  • Python venv  │  • conda 包  │  • 临时文件 │
│  • GGUF 文件    │  • Git 仓库     │  • torch 缓存│  • 日志    │
│  • 模型下载缓存  │  • 实验数据     │  • 数据集缓存 │  • 备份    │
└─────────────────┴─────────────────┴─────────────┴───────────┘
```

**关键设计决策：**
- **模型独立分区（D:）**：大文件集中管理，避免碎片，方便备份
- **缓存独立分区（F:）**：可随时清理，不影响系统和项目
- **Swap 分区（G:）**：小容量高频读写，NTFS 压缩节省空间

## 四、WSL2 配置

### 4.1 全局配置（`%USERPROFILE%\.wslconfig`）

```ini
[wsl2]
# 内存分配（总内存 32GB，分配 12GB 给 WSL2）
memory=12GB

# CPU 核心数（i5-10400F 共 12 线程，分配 8 个）
processors=8

# 交换文件大小
swap=4GB

# 交换文件位置（放在 G 盘）
swapFile=G:\\wsl-swap.vhdx

# localhost 端口转发（启用，方便 Windows 访问 WSL 服务）
localhostForwarding=true

# 网络模式（默认 NAT，如需桥接可改为 mirrored）
# networkingMode=NAT

# 自动挂载 Windows 磁盘
[automount]
enabled=true
mountFsTab=true
root=/mnt/
options="metadata,umask=22,fmask=11"
```

**生效方式：** `wsl --shutdown` 后重新启动 WSL2

### 4.2 便捷目录软链接

```bash
# 创建雅典娜工作目录结构
mkdir -p ~/ai/{models,workspace,cache,logs,backup}

# 建立软链接（WSL 路径 → Windows 分区）
ln -sf /mnt/d ~/ai/models
ln -sf /mnt/e ~/ai/workspace
ln -sf /mnt/f ~/ai/cache
ln -sf /mnt/g ~/ai/logs

# 验证
ls -la ~/ai/
# 输出：
# models -> /mnt/d
# workspace -> /mnt/e
# cache -> /mnt/f
# logs -> /mnt/g
```

## 十、故障速查

| 症状 | 可能原因 | 解决方案 |
| :--- | :--- | :--- |
| WSL2 启动失败 | 内存分配过大 | 减小 `.wslconfig` 中的 `memory` 值 |
| Ollama 无法识别 GPU | CUDA 驱动问题 | 运行 `nvidia-smi` 检查驱动，重装 CUDA Toolkit |
| 模型下载慢 | 网络问题 | 检查代理设置，或切换 HuggingFace 镜像 |
| 磁盘空间不足 | 缓存膨胀 | 清理 F 盘：`rm -rf /mnt/f/pip/*`, `rm -rf /mnt/f/conda/*` |
| Gateway 无法启动 | 端口占用 | 检查 18789 端口：`lsof -i :18789`，结束占用进程 |
| Python 包安装失败 | 权限问题 | 确保使用虚拟环境，避免系统级 pip |
